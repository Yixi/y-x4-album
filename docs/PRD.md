## 1. 产品愿景

### 1.1 为什么要做电子相册？

E-Ink 电子墨水屏具有传统屏幕无法比拟的独特优势：极低功耗、阳光下可读、无蓝光护眼、常显不耗电。这些特性让 E-Ink 设备天然适合作为"数字相框"——一种可以随时更换内容、不需要充电线束缚的照片展示设备。

然而，市面上大多数电子墨水设备主要面向阅读场景（电子书、笔记），缺乏专为图片浏览优化的固件。Xteink X4 作为一款 800x480 分辨率的 E-Ink 设备，拥有足够的显示面积来呈现精美的照片，但需要一个专门的相册固件来释放这一潜力。

### 1.2 目标用户

| 用户画像 | 描述 | 核心需求 |
|----------|------|----------|
| **桌面摆件爱好者** | 希望桌面有一个静谧的照片展示设备 | 幻灯片自动播放、省电、美观 |
| **旅行记忆者** | 希望随身携带精选照片，随时翻阅 | 流畅浏览、目录组织、便携 |
| **摄影爱好者** | 欣赏黑白/灰度照片的独特质感 | 灰度渲染质量、大图支持 |
| **极简主义者** | 厌倦发光屏幕，追求纸质般的视觉体验 | 护眼、无干扰、持久续航 |
| **X4 设备用户** | 已拥有 X4，希望拓展设备用途 | 简单刷入、兼容现有 TF 卡内容 |

### 1.3 解决的痛点

1. **E-Ink 图片浏览体验差**：现有 E-Ink 阅读器虽然可以显示图片（如书籍封面），但缺乏专门的图片浏览、管理和展示功能
2. **数字相框功耗过高**：传统 LCD/LED 数字相框需要持续供电，电池续航极短
3. **照片整理不便**：用户将照片放入 TF 卡后，缺乏直观的浏览和组织方式
4. **灰度照片欣赏**：E-Ink 的灰度抖动渲染能赋予照片独特的艺术质感，但需要专门的渲染优化

### 1.4 产品定位

> **Y-X4-Album** 是一款面向 Xteink X4 E-Ink 设备的开源电子相册固件，提供流畅的图片浏览体验，支持 TF 卡图片读取、高质量灰度渲染和直觉式按钮交互。它让你的 E-Ink 设备变成一个优雅的、超长续航的随身相册和桌面相框。

## 2. 核心功能规格

### 2.1 图片浏览

#### 2.1.1 支持的图片格式

| 格式 | 支持状态 | 说明 |
|------|----------|------|
| **JPEG** (.jpg/.jpeg) | ✅ 优先支持 | 使用 JPEGDEC 库解码，最常见的照片格式 |
| **PNG** (.png) | ✅ 支持 | 使用 PNGdec 库解码，支持透明通道（透明映射为白色） |
| **BMP** (.bmp) | ✅ 支持 | 原生位图，直接渲染，无需解码 |

#### 2.1.2 支持的分辨率范围

| 参数 | 值 | 说明 |
|------|-----|------|
| **最小分辨率** | 16 × 16 px | 小于此值的图片忽略（可能是图标/缩略图） |
| **推荐分辨率** | 800 × 480 px | 与屏幕完全匹配，无需缩放 |
| **最大安全分辨率** | 4096 × 4096 px | 超过此尺寸警告解码可能缓慢 |
| **硬性上限** | 8192 × 8192 px | 超过此尺寸跳过不加载（内存安全） |

#### 2.1.3 大图处理策略

由于 ESP32-C3 仅有 ~380KB RAM，无法将大图完整加载到内存，必须采用分块处理：

1. **解码降采样**：JPEG 解码时利用 JPEGDEC 的内置降采样功能（1/2、1/4、1/8），在解码阶段直接缩小到接近显示尺寸
2. **分块解码**：PNG 使用扫描线式解码，每次仅处理一行像素，不需要完整图片缓冲
3. **适应屏幕**：所有图片最终映射到 800×480（横屏）或 480×800（竖屏）的显示区域
4. **缩放模式**：
   - **适应**（Fit，默认）：保持比例，完整显示，上下或左右留黑/白边
   - **填充**（Fill）：保持比例，裁切超出部分，无留白
   - **拉伸**（Stretch）：变形填满屏幕

#### 2.1.4 图片渲染流程

```
TF卡图片文件 → 格式检测 → 解码器选择 → 分块解码 → 灰度转换 → 抖动处理 → 帧缓冲 → E-Ink刷新
```

#### 2.1.5 边界条件和错误处理

| 场景 | 处理方式 |
|------|----------|
| 图片文件损坏 | 显示错误占位图（❌ 图标 + 文件名），跳转到下一张 |
| 格式不支持 | 扫描时跳过，不在图库中显示 |
| 解码内存不足 | 尝试更高降采样率，仍失败则显示错误提示 |
| 文件读取失败 | 提示"读取失败"，允许用户重试或跳过 |
| 图片文件名包含特殊字符 | 使用 UTF-8 长文件名支持（USE_UTF8_LONG_NAMES） |
| 0字节文件 | 扫描时跳过 |
| TF卡中途拔出 | 检测到存储异常，返回主页显示"请插入TF卡" |

### 2.2 图库 / 网格视图

图库是用户进入应用后看到的主界面，以缩略图网格形式展示当前目录下的所有图片。

#### 2.2.1 缩略图规格

| 参数 | 值 | 说明 |
|------|-----|------|
| **网格布局** | 4列 × 3行 | 横屏模式下每页显示 12 张缩略图 |
| **缩略图尺寸** | 约 180 × 140 px | 含间距和选中框 |
| **缩略图格式** | 1-bit BMP | 缓存为单色位图，最小化存储 |
| **缓存位置** | `/.y-x4-album/thumbs/<path_hash>.bmp` | TF 卡缓存目录 |

#### 2.2.2 缩略图生成策略

1. **首次扫描**：进入图库时扫描当前目录，发现新图片时后台生成缩略图
2. **增量生成**：仅为没有缓存的图片生成缩略图，已有缓存的直接加载
3. **占位显示**：缩略图未就绪时显示加载占位图（灰色方块 + 文件名前几个字符）
4. **优先级**：优先生成当前可视页面的缩略图，后台异步生成其余页面

#### 2.2.3 导航

| 操作 | 按钮 | 说明 |
|------|------|------|
| **移动选中框** | Up/Down/Left/Right | 在网格中移动高亮框 |
| **打开图片** | Confirm | 进入全屏浏览模式 |
| **翻页** | PageForward/PageBack | 上一页/下一页缩略图 |
| **返回** | Back | 返回文件浏览器 / 上级目录 |

#### 2.2.4 分页规则

- 每页 12 张缩略图（4×3）
- 页码指示器显示在底部状态栏：`第 1/5 页 · 共 58 张`
- 翻页后焦点默认在新页面的第一张（向后翻）或最后一张（向前翻）
- 空目录显示提示："此文件夹没有图片"

#### 2.2.5 排序方式

| 排序选项 | 说明 |
|----------|------|
| **文件名升序**（默认） | A-Z，数字按自然排序（img2 < img10） |
| **文件名降序** | Z-A |
| **修改时间最新** | 最新修改的图片在前 |
| **修改时间最早** | 最早修改的图片在前 |
| **文件大小** | 从大到小 |

### 2.3 全屏图片浏览

全屏模式是核心体验——以最佳质量在 E-Ink 屏幕上展示单张图片。

#### 2.3.1 灰度渲染

| 参数 | 值 |
|------|-----|
| **灰度级别** | 4 级灰度（白、浅灰、深灰、黑），通过 4×4 Bayer 矩阵抖动实现 |
| **渲染模式** | 单色模式（BW）/ 灰度模式（Grayscale），用户可切换 |
| **默认模式** | 灰度模式（图片浏览场景下灰度效果更佳） |

灰度渲染流程：
1. 调用 `renderer.storeBwBuffer()` 保存当前黑白缓冲
2. 在灰度缓冲中绘制图片
3. 使用灰度 LUT 刷新 E-Ink
4. 调用 `renderer.restoreBwBuffer()` 恢复黑白缓冲

#### 2.3.2 图片切换

| 操作 | 按钮 | 说明 |
|------|------|------|
| **上一张** | Left / PageBack | 切换到上一张图片 |
| **下一张** | Right / PageForward | 切换到下一张图片 |
| **返回图库** | Back | 返回图库网格视图 |
| **切换渲染模式** | Confirm（短按） | 在灰度/单色模式间切换 |
| **图片信息** | Confirm（长按） | 显示图片信息覆盖层 |

#### 2.3.3 图片信息覆盖层

长按 Confirm 时在屏幕底部显示半透明信息条：

```
┌──────────────────────────────────────────┐
│                                          │
│              [图片内容]                    │
│                                          │
├──────────────────────────────────────────┤
│ photo_001.jpg  │ 3024×4032  │ 2.1MB      │
│ 第 15/58 张    │ JPEG       │ 灰度模式    │
└──────────────────────────────────────────┘
```

包含：文件名、原始分辨率、文件大小、当前位置（第 N/总数 张）、格式、当前渲染模式。

#### 2.3.4 缩放和平移（v1.0 功能）

| 参数 | 值 |
|------|-----|
| **缩放级别** | 1×（适应屏幕）、2×、4× |
| **缩放操作** | Up（放大）/ Down（缩小） |
| **平移操作** | 缩放状态下 Left/Right/Up/Down 平移视口 |
| **重置** | Back 键重置为 1× 并返回适应模式 |

缩放实现约束：
- 缩放时重新解码图片的对应区域（不缓存整张大图）
- 使用 JPEG 区域解码特性（MCU 块对齐）提高效率
- 每次平移步长 = 屏幕尺寸的 50%

#### 2.3.5 边界条件

| 场景 | 处理方式 |
|------|----------|
| 第一张图片按"上一张" | 循环到最后一张（可在设置中关闭循环） |
| 最后一张图片按"下一张" | 循环到第一张 |
| 图片比例与屏幕差异大 | 根据缩放模式处理（Fit/Fill/Stretch） |
| 竖屏图片在横屏模式下 | 左右留白，居中显示 |
| 横屏图片在竖屏模式下 | 上下留白，居中显示 |

### 2.4 幻灯片模式

自动播放模式，适合作为桌面相框使用。

#### 2.4.1 播放控制

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| **播放间隔** | 30 秒 | 5s / 10s / 30s / 1min / 5min / 15min / 30min / 1h | 每张图片展示时长 |
| **播放顺序** | 顺序 | 顺序 / 随机 / 随机不重复 | 图片切换顺序 |
| **循环模式** | 开启 | 开/关 | 播放完所有图片后是否重新开始 |
| **过渡效果** | 直接切换 | 直接切换 / 渐入 | E-Ink 限制，仅支持有限过渡 |

#### 2.4.2 操作

| 操作 | 按钮 | 说明 |
|------|------|------|
| **暂停/继续** | Confirm | 切换暂停/播放状态 |
| **上一张** | Left / PageBack | 手动切回上一张，暂停自动播放 |
| **下一张** | Right / PageForward | 手动跳到下一张，暂停自动播放 |
| **退出幻灯片** | Back | 返回到全屏浏览模式 |
| **调整间隔** | Up/Down | 长按增加/减少播放间隔 |

#### 2.4.3 状态显示

- 播放状态指示器（屏幕右上角小图标）：▶ 播放中 / ⏸ 已暂停
- 暂停时显示剩余时间和当前位置
- 5 秒无操作后状态指示器自动消失（不影响图片显示）

#### 2.4.4 幻灯片范围

| 范围选项 | 说明 |
|----------|------|
| **当前文件夹** | 仅播放当前目录下的图片 |
| **当前文件夹及子目录** | 递归扫描所有子目录中的图片 |
| **所有图片** | 从根目录递归扫描所有图片 |

#### 2.4.5 幻灯片电源优化

- 在两次图片切换之间，CPU 进入低功耗模式（10MHz）
- 定时器唤醒 CPU 进行下一次图片切换
- 幻灯片模式下自动睡眠超时延长为正常值的 3 倍（避免相框场景频繁睡眠）
- 电池电量低于 5% 时暂停幻灯片并提示充电

### 2.5 文件管理

#### 2.5.1 文件浏览器

提供类似文件管理器的界面，让用户在 TF 卡目录结构中导航。

**布局**：列表视图，每行显示一个文件/文件夹。

| 列 | 内容 |
|-----|------|
| 图标 | 📁 文件夹 / 🖼 图片文件 |
| 名称 | 文件/文件夹名（过长时截断，尾部显示 ...） |
| 信息 | 文件夹：包含图片数量；文件：文件大小 |

#### 2.5.2 目录导航

| 操作 | 按钮 | 说明 |
|------|------|------|
| **上下移动** | Up/Down | 在列表中选择 |
| **进入目录** | Confirm | 打开选中的文件夹 |
| **打开图片** | Confirm | 以全屏模式打开选中的图片 |
| **返回上级** | Back | 返回父目录 |
| **快速翻页** | PageForward/PageBack | 列表快速翻页 |

#### 2.5.3 根目录设置

用户可以指定一个 TF 卡上的目录作为"相册根目录"：

- **默认根目录**：TF 卡根目录 `/`
- **自定义根目录**：用户在文件浏览器中导航到目标目录，通过菜单设为根目录
- **效果**：设定后，图库视图和幻灯片默认从此目录开始
- **存储**：根目录路径保存在设置文件中

#### 2.5.4 文件过滤

- 文件浏览器中默认仅显示：
  - 文件夹（隐藏文件夹除外，即以 `.` 开头的）
  - 支持的图片文件（.jpg, .jpeg, .png, .bmp）
- 文件夹排列在图片前面
- 可在设置中切换"显示所有文件"模式

#### 2.5.5 边界条件

| 场景 | 处理方式 |
|------|----------|
| 空目录 | 显示"此文件夹为空" |
| 目录中无图片 | 显示"此文件夹没有图片" |
| 深层嵌套目录（>10 层） | 允许继续导航，但路径栏截断显示 |
| 文件/文件夹数量 > 1000 | 分页加载，每页 20 项 |
| TF 卡未插入 | 显示"请插入 TF 卡"引导页 |
| TF 卡文件系统损坏 | 显示"存储卡异常，请检查"提示 |

#### 2.6.2 设置持久化

- **存储格式**：二进制文件 `settings.bin`，位于 TF 卡根目录的 `.y-x4-album/` 目录
- **保存时机**：每次设置变更后立即写入（防止意外断电丢失）
- **加载时机**：系统启动时从 TF 卡读取，文件不存在则使用默认值
- **版本兼容**：设置文件包含版本号，升级固件时自动迁移旧格式
- **校验**：CRC 校验防止文件损坏，校验失败则恢复默认值

## 5. E-Ink 特性对产品的影响

### 5.1 刷新延迟

**影响**：E-Ink 完全刷新需要 ~1.7 秒，快速刷新也需要 200-500ms。

**产品应对策略**：
1. **缓冲按键输入**：在 E-Ink 刷新期间接收但缓冲用户的按键操作，刷新完成后立即处理
2. **"正在加载"状态**：图片解码和刷新期间在状态栏显示加载指示
3. **智能刷新模式选择**：
   - 图片切换使用快速刷新（速度优先）
   - 每 N 次切换执行一次完全刷新（清除残影）
   - 菜单操作使用快速刷新
4. **预加载**：在用户浏览当前图片时，后台预解码下一张图片（如果内存允许）

### 5.2 灰度限制

**影响**：E-Ink 仅支持 4 级灰度（通过 Bayer 抖动模拟），远低于照片的 256 级灰度。

**产品应对策略**：
1. **高质量抖动算法**：使用 4×4 Bayer 矩阵抖动，在有限灰度级别下最大化视觉效果
2. **双模式切换**：允许用户在灰度模式和纯黑白模式间切换，某些高对比度照片在纯黑白下效果更好
3. **对比度优化**：解码时可选择增强对比度/锐度，补偿灰度级别不足
4. **用户预期管理**：首次使用时展示示例效果，让用户了解 E-Ink 灰度的风格和局限

### 5.3 残影处理

**影响**：E-Ink 快速刷新会导致前一帧的内容残留（残影/鬼影）。

**产品应对策略**：
1. **定期完全刷新**：用户可设置每 N 张图片执行一次完全刷新（默认每 5 张）
2. **强制刷新快捷键**：提供手动触发完全刷新的操作（设置中可配置为某个按钮组合）
3. **菜单回退刷新**：从菜单/设置返回图片浏览时自动执行完全刷新
4. **反转闪烁**：完全刷新时先全屏反转再恢复，最大化清除残影

### 5.4 常显特性

**影响**：E-Ink 断电后仍能保持最后一帧画面显示。

**产品利用方式**：
1. **休眠画面**：休眠时保留当前图片，实现"始终展示"的相框效果
2. **超长待机**：深度睡眠时屏幕仍显示图片，仅消耗极低的 RTC 电流
3. **关机展示**：关机前可选择显示一张指定图片，作为"静态展示"

### 5.5 温度敏感性

**影响**：E-Ink 刷新速度和质量受温度影响。

**产品应对策略**：
- 低温环境下自动降低刷新速度以保证显示质量
- 极端温度下提示用户（如果设备有温度传感器）

## 6. 图片格式兼容性矩阵

### 6.1 JPEG 格式

| 特性 | 支持状态 | 说明 |

| PNG-24（真彩色） | ✅ 支持 | RGB 24-bit |
| PNG-32（真彩色+Alpha） | ✅ 支持 | Alpha 通道映射为白色背景 |
| 灰度 PNG | ✅ 支持 | 直接映射到 E-Ink 灰度 |

| 特性 | 支持状态 | 说明 |
|------|----------|------|
| 1-bit 单色 | ✅ 支持 | 直接位图映射 |
| 4-bit 索引 | ✅ 支持 | 16 色调色板 |
| 8-bit 索引 | ✅ 支持 | 256 色调色板 |
| 24-bit 真彩 | ✅ 支持 | RGB 无压缩 |

| RLE 压缩 | ❌ 不支持 | 仅支持无压缩 BMP |

| 格式 | 状态 | 原因 |

| SVG | ❌ | 矢量渲染在 E-Ink 上无意义 |
| RAW (CR2/NEF/ARW) | ❌ | 文件巨大，解码复杂 |

## 附录

### A. 术语表

| 术语 | 定义 |
|------|------|
| **E-Ink** | 电子墨水显示技术，靠反射光显示，断电保持画面 |
| **帧缓冲（Framebuffer）** | 显示内存，800×480÷8 = 48,000 字节 |
| **Bayer 抖动** | 一种有序抖动算法，用少量灰度级别模拟更多灰度层次 |
| **完全刷新（Full Refresh）** | E-Ink 先全黑再全白再显示内容，耗时~1.7s，无残影 |
| **快速刷新（Fast Refresh）** | 直接更新变化部分，速度快但可能有残影 |
| **Activity** | UI 界面单元，遵循生命周期模式（onEnter/loop/render/onExit） |
| **HAL** | 硬件抽象层，屏蔽底层硬件差异 |
| **MCU** | 微控制器单元，本项目使用 ESP32-C3 |
| **降采样** | 在解码阶段缩小图片分辨率，节省内存 |
