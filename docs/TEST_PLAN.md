## 3. 代码审查检查清单

### 3.1 内存安全
- [ ] 所有 `malloc`/`new` 都有 `nullptr` 检查
- [ ] 每个 `malloc` 有对应的 `free`，每个 `new` 有对应的 `delete`
- [ ] 无 `std::shared_ptr` 使用（项目禁止）
- [ ] 热路径无 `std::string` / `String` 使用
- [ ] 栈局部变量 < 256 字节
- [ ] 无内存泄漏路径（所有提前 return 都释放资源）

### 3.2 Activity 生命周期
- [ ] `onEnter()` 中分配的资源在 `onExit()` 中释放
- [ ] FreeRTOS 任务在 `onExit()` 中 `vTaskDelete()`
- [ ] 文件句柄在 `onExit()` 中关闭
- [ ] RenderLock 正确传递给 `render()` 方法
- [ ] Activity 栈操作正确（push/pop/replace）

### 3.3 RISC-V 对齐合规
- [ ] 无 `uint8_t*` 强转为 `uint16_t*`/`uint32_t*` 等宽类型
- [ ] 多字节数据访问使用 `memcpy` 而非指针强转
- [ ] 结构体无隐式对齐假设

### 3.4 线程安全
- [ ] SD 卡访问通过 `HalStorage`（带 mutex）
- [ ] 渲染操作持有 `RenderLock`
- [ ] 无跨线程裸指针共享

### 3.5 资源管理
- [ ] 大缓冲区（>256B）使用 `malloc`，非栈分配
- [ ] RAII 模式用于 mutex/lock 管理
- [ ] 图片解码分块进行，非整张加载

### 5.2 运行时堆分配预算
| 组件 | 大小 | 说明 |
|------|------|------|
| ImageIndex (500条) | ~70,000 B | `malloc(500 * sizeof(ImageEntry))` |
| PNG 解码器 (堆分配后) | ~58,000 B | PNGIMAGE 结构体（**当前未修复，仍在栈上**） |
| JPEG 解码器 (堆分配后) | ~17,900 B | JPEGIMAGE 结构体（**当前未修复，仍在栈上**） |
| AtkinsonDitherer | ~4,800 B | 3行误差缓冲 (800px宽) |
| BW buffer 备份 (灰度模式) | ~48,000 B | storeBwBuffer 临时分配 |
| **峰值总计 (灰度 PNG)** | **~181KB** | 索引 + PNG + ditherer + BW backup |
| **可用堆** | **~163KB** | ⚠️ **超出！需降低 maxEntries 或优化** |

### 5.2 运行时内存检查点
- [ ] 启动后空闲状态：`ESP.getFreeHeap()` > 200KB
- [ ] 打开图库（加载缩略图后）：`ESP.getFreeHeap()` > 100KB
- [ ] 全屏浏览（解码图片中）：`ESP.getFreeHeap()` > 50KB
- [ ] 幻灯片连续播放 100 张后：`ESP.getFreeHeap()` > 50KB
- [ ] 多次 Activity 进入/退出后无内存泄漏
